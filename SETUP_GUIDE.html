When the Nagarkot Network Facebook Page publishes a new post, Facebook instantly notifies your Supabase Edge Function via a webhook. The Edge Function validates the request and saves the post as a news article in your Supabase database — where it immediately appears on your website.

ℹ️
Important: This only works with a Facebook Page, not a personal profile. Make sure Nagarkot Network has a Facebook Page (which it already does based on the website footer).
1

Update your Supabase database table

First, add two new columns to your news table so we can track which posts came from Facebook and avoid duplicates.

Go to your Supabase Dashboard → SQL Editor and run this:

-- Add Facebook tracking columns to your news table
ALTER TABLE news
  ADD COLUMN IF NOT EXISTS source TEXT DEFAULT 'manual',
  ADD COLUMN IF NOT EXISTS fb_post_id TEXT UNIQUE,
  ADD COLUMN IF NOT EXISTS fb_post_url TEXT;

-- Create an index for fast duplicate checks
CREATE INDEX IF NOT EXISTS idx_news_fb_post_id ON news(fb_post_id);
CREATE INDEX IF NOT EXISTS idx_news_source ON news(source);
2

Create a Facebook Developer App

1
Go to developers.facebook.com and log in with the Page owner's account.
2
Click "My Apps" → "Create App". Choose "Business" as the app type.
3
Give it a name like "Nagarkot Network Sync" and click Create App.
4
From the App Dashboard, go to Settings → Basic. Copy and save your App ID and App Secret — you'll need these.
5
In the left menu, find Products and click "+" next to Webhooks to add it to your app.
⚠️
Keep your App Secret private! Never put it in your HTML files or commit it to GitHub. We'll store it safely in Supabase secrets.
3

Deploy the Supabase Edge Function

The Edge Function file is included in your download at supabase/functions/facebook-webhook/index.ts. Follow these steps to deploy it.

3a. Install Supabase CLI

# Install Supabase CLI (if not already installed)
npm install -g supabase

# Log in to your Supabase account
supabase login

3b. Set up environment secrets

These secrets are stored securely in Supabase and never exposed publicly:

# Set your Facebook App Secret (from Step 2)
supabase secrets set FB_APP_SECRET=your_facebook_app_secret_here

# Set a verify token — make up ANY random string (e.g. "nagarkot2026")
supabase secrets set FB_VERIFY_TOKEN=nagarkot2026

# Your Supabase service role key (from Supabase Dashboard → Settings → API)
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here
📋 Summary of secrets you need
Secret NameWhere to find it
FB_APP_SECRETFacebook Developer App → Settings → Basic → App Secret
FB_VERIFY_TOKENMake up any random string (e.g. nagarkot2026abc)
SUPABASE_SERVICE_ROLE_KEYSupabase Dashboard → Project Settings → API → service_role key

3c. Deploy the function

# Link your project (replace with your Supabase project ID)
supabase link --project-ref waqfebkbgyfhybompnsm

# Deploy the Facebook webhook function
supabase functions deploy facebook-webhook

After deployment, your webhook URL will be:

https://waqfebkbgyfhybompnsm.supabase.co/functions/v1/facebook-webhook

Save this URL — you'll need it in the next step.

4

Register the Webhook in Facebook

1
In your Facebook App dashboard, go to Webhooks (under Products in the left sidebar).
2
Select "Page" from the dropdown and click "Subscribe to this object".
3
Fill in the fields:
• Callback URL: https://waqfebkbgyfhybompnsm.supabase.co/functions/v1/facebook-webhook
• Verify Token: The same token you set in Step 3b (e.g. nagarkot2026)
4
Click "Verify and Save". Facebook will call your URL to confirm it works. ✅
5
After verifying, check the "feed" subscription field and click Subscribe.
6
Now go to Facebook Login → Settings and add your webhook URL to the Deauthorize Callback URL and Valid OAuth Redirect URIs fields.
✅
If Facebook shows "Verified" — your Edge Function is deployed correctly and the connection is working!
5

Connect your Facebook Page to the App

To receive posts from your specific Page, you need to subscribe to it using the Graph API.

Get a Page Access Token

1
Go to developers.facebook.com/tools/explorer (Graph API Explorer).
2
Select your App from the dropdown. Click "Generate Access Token".
3
Select these permissions: pages_read_engagement, pages_manage_posts.
4
Copy the token. Then run this in Graph API Explorer to subscribe your page:
POST /{your-page-id}/subscribed_apps

# Parameters:
subscribed_fields=feed
access_token=YOUR_PAGE_ACCESS_TOKEN
💡
Find your Page ID: Go to your Facebook Page → About → scroll down to see Page ID, or use Graph API: GET /me?fields=id,name
6

Submit for Facebook App Review

⏳
This step is required for a public Page. Facebook requires App Review before you can receive live data from pages with many followers. This process takes 1–7 business days.
1
In your App dashboard, go to App Review → Permissions and Features.
2
Request pages_read_engagement permission. Explain that you're the Page owner and want to sync posts to your website.
3
Add a short screencast showing how the integration works (you can use Loom or a simple screen recording).
4
Submit for review. Facebook will email you when approved. Until then, you can test with your own personal account as a test user.
🎉
Once approved: Every post on the Nagarkot Network Facebook Page will automatically appear on nagarkotnetwork.com within seconds!
🧪

Test it without App Review

While waiting for App Review, you can test the webhook by manually sending a test payload to your Edge Function:

# Test your Edge Function directly (simulates a Facebook post)
curl -X POST \
  https://waqfebkbgyfhybompnsm.supabase.co/functions/v1/facebook-webhook \
  -H "Content-Type: application/json" \
  -d '{
    "object": "page",
    "entry": [{
      "changes": [{
        "field": "feed",
        "value": {
          "item": "post",
          "message": "नगरकोट नेटवर्कको परीक्षण समाचार\nयो एक परीक्षण पोस्ट हो जुन फेसबुकबाट आएको हो।",
          "post_id": "test_123456",
          "created_time": 1700000000,
          "permalink_url": "https://facebook.com/test"
        }
      }]
    }]
  }'

Note: The signature verification will fail for manual tests. You can temporarily comment out the signature check in the Edge Function for testing, then re-enable it.

Ready to set up? 🚀

Your Edge Function code is ready to deploy. Follow the steps above and your Facebook posts will auto-publish to nagarkotnetwork.com.
